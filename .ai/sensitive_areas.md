# Sensitive Areas

Changes in these areas require extra caution:
- identity/auth
- billing/payments
- booking logic
- scheduling rules
- permissions and lesson access
- audit/operational logging

## Required Change Notes
1. Explain current behavior.
2. Explain intended change.
3. Describe regression/security risk.
4. Describe verification plan (tests/checks).

Avoid refactoring sensitive areas without a direct task requirement.
