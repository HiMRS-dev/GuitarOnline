## Migration Notes

1. Deploy latest ref/tag and run DB migrations:
   - `docker compose -f docker-compose.prod.yml exec -T app alembic upgrade head`
2. Verify migration state:
   - `docker compose -f docker-compose.prod.yml exec -T app alembic current`
3. Run smoke and load sanity checks:
   - `docker compose -f docker-compose.prod.yml exec -T app python scripts/deploy_smoke_check.py`
   - `docker compose -f docker-compose.prod.yml exec -T app python scripts/load_sanity.py`

## Rollback Notes

1. Redeploy previous stable ref/SHA:
   - use `.github/workflows/deploy.yml` with `ref=<previous_sha_or_tag>`.
2. Restore DB from backup if migration/data rollback is required:
   - `powershell -ExecutionPolicy Bypass -File scripts/db_restore.ps1 -InputFile <backup.sql>`
3. Re-run smoke checks on rolled-back version:
   - `docker compose -f docker-compose.prod.yml exec -T app python scripts/deploy_smoke_check.py`
